# Testing Overview

## Test Framework and Setup

The project uses two test frameworks targeting different layers:

| Layer | Framework | Version |
|-------|-----------|---------|
| Frontend (unit + component) | Vitest | ^4.1.5 |
| Frontend (visual smoke) | Playwright | ^1.59.1 |
| Rust backend | cargo test (inline `#[cfg(test)]` modules) | stable |

Frontend tests run under a jsdom environment with React Testing Library (`@testing-library/react ^16.3.0`) and `@testing-library/jest-dom ^6.9.1` for DOM matchers.

### Global Test Setup

`src/test-setup.ts` is loaded before every test file and provides:

- `@testing-library/jest-dom` DOM matchers
- Forced `en_US.UTF-8` locale for consistent formatting across environments
- `@tauri-apps/api/core` (`invoke`) and `@tauri-apps/api/event` (`listen`, `emit`) mocked to prevent IPC calls hitting the native layer
- `ResizeObserver` stub for components that observe element dimensions
- `window.matchMedia` stub for responsive/theme logic

No additional setup is required before running tests — install dependencies first.

## Running Tests

### Unit and Component Tests

```bash
# Run full suite once
bun run test

# Watch mode (re-runs on file change)
bun run test:watch

# Run with coverage report
bun run test:coverage
```

Vitest scans `src/**/*.{test,spec}.{ts,tsx}` and excludes `node_modules`, `.kspec-worktrees`, and `.worktrees`.

### Visual Smoke Tests (Playwright)

```bash
# Requires Chromium — install once
bunx playwright install --with-deps chromium

# Run visual smoke suite
bun run test:visual
```

Playwright tests live in `tests/visual/`. The current suite (`hud-agent-smoke.spec.ts`) validates HUD CSS animations, glow effects, and agent panel transitions in a Chromium browser.

### Rust Backend Tests

```bash
# Run Rust unit tests from project root
cd src-tauri && cargo test
```

Rust tests are inline `#[cfg(test)]` modules. Tested modules include `session_manager`, `health_manager`, `orchestration`, `project_scanner`, `upgrade_manager`, `database`, and `ai_runtime` (semantic cache, context compressor, opt metrics).

### Touched-Folder Coverage Gate

```bash
bun run coverage:touched
```

This script (`scripts/check-touched-coverage.mjs`) inspects `coverage/coverage-summary.json` and enforces higher per-folder thresholds on critical folders that were modified in the current diff. Run `bun run test:coverage` first to generate the summary. In CI this gate runs automatically on every push/PR.

## Writing New Tests

### File Naming Convention

| Pattern | Usage |
|---------|-------|
| `src/**/*.test.ts` | Pure logic, hooks, stores, IPC layer |
| `src/**/*.test.tsx` | React components |
| `src/**/__tests__/*.test.ts` | Alternative placement (themes, etc.) |
| `tests/visual/*.spec.ts` | Playwright visual tests |

Co-locate unit and component tests next to their source file (e.g., `AgentBoard.test.tsx` beside `AgentBoard.tsx`). The `__tests__/` subdirectory is reserved for cross-cutting concerns like theme validation.

### Test Helpers

There is no shared test-helper module. Common patterns used across test files:

- Mock Tauri IPC calls via `vi.fn()` on the global mock established in `test-setup.ts`
- Use `@testing-library/react` `render` + `screen` + `userEvent` for component tests
- Wrap zustand store resets in `beforeEach` to prevent state leakage between tests

## CI Integration

Workflow: `.github/workflows/ci.yml` — triggers on push to `main` and on pull requests targeting `main`.

**Frontend job** (runs on `ubuntu-latest`):
1. Install dependencies — `bun install --frozen-lockfile`
2. Type check — `bunx tsc --noEmit`
3. Vitest coverage — `bun run test:coverage`
4. Touched-folder coverage gate — `bun run coverage:touched`
5. Coverage diff comment (PRs only) — posts a coverage summary comment via `davelosert/vitest-coverage-report-action@v2`
6. Lint (non-blocking) — `bunx biome check src/` (failures do not fail the build)
7. Playwright visual smoke — `bun run test:visual` (Chromium installed first)
8. Frontend build verification — `bun run build:frontend`

**Rust job** (runs on `ubuntu-latest`):
1. `cargo check` — verifies compilation in `src-tauri/`
2. `cargo clippy -- -D warnings` — linting with warnings treated as errors

The Rust job does not run `cargo test` in CI — Rust tests are run locally. The Tauri desktop build itself is not compiled in CI (requires platform-specific native deps and signing).
